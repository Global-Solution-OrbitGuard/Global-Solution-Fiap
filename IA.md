# AI.md

Projeto: OrbitGuard

Interação 1

O que foi solicitado para a IA?

Solicitação de orientação para o desenvolvimento da plataforma OrbitGuard, incluindo arquitetura do sistema, integração com dados climáticos e utilização de APIs gratuitas.

O que a IA retornou?
Foi sugerida uma arquitetura composta por:

- Landing Page em HTML, CSS e JavaScript;
- Dashboard para monitoramento climático;
- Utilização da API Open-Meteo para dados meteorológicos;
- Utilização da NASA EONET para monitoramento de eventos ambientais;
- Integração futura com sensores Arduino;
- Estrutura de pastas e organização do projeto.

O que foi alterado ou rejeitado e o motivo?

A arquitetura proposta foi adaptada para atender aos requisitos específicos do nosso projeto e das disciplinas da GS.

Interação 3

O que foi solicitado para a IA?
Orientação sobre como integrar dados climáticos reais e sensores Arduino ao sistema.

O que a IA retornou
Foi explicado o fluxo de funcionamento:

1. APIs meteorológicas fornecem dados climáticos.
2. O sistema processa os dados.
3. O Arduino coleta informações locais através de sensores.
4. Os dados são exibidos na plataforma em forma de alertas e indicadores.

O que foi alterado ou rejeitado e o motivo?

A implementação será simplificada devido às limitações de hardware e tempo de desenvolvimento.
